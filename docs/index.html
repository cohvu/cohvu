How it works

Two tools. read and contribute. That's the entire interface between your agents and Cohvu.

Before starting work, every agent calls read. It gets back everything the project knows — decisions made, constraints discovered, patterns established, directions chosen. Not a dump of everything. The right things, retrieved by meaning, ranked by relevance to what the agent is about to do.

As the agent works, it calls contribute. Not at the end. Not as a summary. In the moment — when it makes a decision, when it discovers something, when it learns something that changes how it understands the problem. Every contribution is immediately available to every other agent on the project.

Run npx cohvu and everything is handled. It detects every AI tool on your machine and configures them all in one pass. Claude Code and Cursor get a plugin. Windsurf, Cline, and Codex are configured automatically. The agents start reading and contributing on their next session. You don't wire anything. You don't configure anything. You just work.

Invite your team and the understanding becomes shared. Admins shape what the project knows and how it grows. Members contribute through their agents automatically — no extra steps, no manual notes. Viewers see everything the agents understand without being able to change it. What one person's agent learns, every person's agent knows. The organizational understanding compounds with every session, across every tool, across every timezone.

The project gets smarter with every session. What starts as a blank slate becomes an accurate, living picture of your codebase — built by the agents themselves, without anyone writing documentation.
