How it works

Create a project. Every agent that connects reads from it before starting work and contributes to it as they go. The project gets richer with every session.

npx cohvu login. One command detects every tool on your machine and configures them all. Claude Code and Cursor get a plugin. Windsurf, Cline, and Codex are configured automatically.

Invite your team and every agent across your entire organization works from the same understanding. What one person's agent learns every person's agent knows. New teammates inherit everything from the first session.

Autonomous agents connect via a secure agent link. Your CI pipelines and scheduled workflows get the same understanding as your interactive sessions.

Browse what your project knows, remove anything that shouldn't be there, switch between projects. Everything else happens on its own.

In practice

Open a fresh context window. The agent already understands your codebase, your decisions, your reasoning, your direction. You don't re-explain anything. You just continue.
Use Claude Code to reason through your architecture. Switch to Codex to debug. Codex already knows every decision you made, every constraint you discovered, every direction you chose. It doesn't just see the code — it understands why the code is the way it is. The debugger has the designer's full context. That has never been possible before.
Two agents working different parts of the same feature simultaneously. One discovers the API rate limits at 100 requests per minute on the backend. The other knows it before writing a single line of frontend code that would have hit that wall. They never talked to each other. They didn't have to.
A new developer joins your team. Their agent inherits everything your agents have ever learned about the project — every architectural decision, every constraint, every dead end, every direction chosen and why. Not documentation nobody wrote. Not comments that went stale. The actual reasoning. On the first session.
Come back after a week away. Your agent already knows where you left off, what you were thinking, what the next step was. You don't spend the first twenty minutes reconstructing context. You just work.
A constraint exists deep in the codebase that's easy to miss. One agent discovers it. Contributes it. No agent ever violates it again. The thing that used to require tribal knowledge is now just known.
Switch between Claude Code, Cursor, Windsurf, Cline, and Codex across your workflow. Every single one understands your project the same way. No agent is the smart one that knows everything. No agent is the one that has no idea what's going on.
Your codebase has conventions. Naming patterns, architectural boundaries, preferred libraries. An agent that has never touched this project before follows them on the first session — because every previous agent contributed them and they're already there waiting.
You're deep in a complex refactor that spans multiple sessions. Each session builds on the last. The agent remembers what it already changed and why. It doesn't suggest reverting something it already decided to change for a reason. The work compounds instead of repeating.
The decision log that actually gets maintained. Not because anyone writes documentation. Because agents contribute decisions as they make them. Six months of architectural history, always current, always available, surfaced exactly when it's relevant.
Your team's collective understanding in one place. What one person's agent learns, every person's agent knows. Knowledge doesn't live in one developer's head and disappear when they go on vacation. It lives in the project. It's always there.
